# Korea Ads Auditor

한국 광고 계정을 빠르게 진단하고, 비개발자도 실행할 수 있는 개선 액션으로 바꾸는 스킬이다. API 연결을 전제로 하지 않는다. 사용자가 제공하는 CSV, 엑셀 요약, 스크린샷, 붙여넣은 지표만으로도 1차 진단을 수행한다. 사용자가 원하면 Codex 브라우저로 이미 로그인된 광고 관리자 화면을 읽기 전용으로 확인해 표, 지표, 화면 상태를 수집한다. 네이버/카카오/쿠팡/당근 같은 국내 플랫폼뿐 아니라 한국 광고주가 자주 함께 운영하는 Google Ads, YouTube, Meta Ads도 1차 플랫폼으로 다룬다.

## Use This First

광고 분석을 시작하기 전에 아래 정보를 한 번에 요청한다. 사용자가 이미 제공한 항목은 다시 묻지 말고 추출해서 사용한다.

1. 업종: 이커머스, 로컬 서비스, B2B, 교육, 병원/의료, 금융, 부동산, 앱, 콘텐츠/인포상품, 기타
2. 월 광고비: 총액과 플랫폼별 대략 비중
3. 주요 목표: 매출, ROAS, 리드, 전화, 예약, 앱설치, 방문, 브랜드 인지도
4. 사용 플랫폼: Google, YouTube, Meta, 네이버, 카카오, 쿠팡, 당근, TikTok, 기타
5. 제공 가능한 데이터: 성과 CSV, 캠페인 표, 검색어/키워드 리포트, 상품별 성과, 스크린샷, 랜딩페이지 URL, 로그인된 광고 관리자 화면

데이터가 부족하면 "진단 불가"로 멈추지 말고, 가정 기반의 예비 진단과 추가로 필요한 데이터 목록을 분리해서 제공한다.

## Workflow

1. **맥락 파악**: 업종, 예산 규모, 목표, 플랫폼 조합을 정리한다.
2. **데이터 정규화**: 플랫폼별 지표명을 공통 용어로 매핑한다.
3. **브라우저 보조 수집**: 사용자가 요청하고 로그인 화면이 준비된 경우, 읽기 전용으로 화면의 표/지표/상태를 확인한다.
4. **플랫폼별 점검**: 관련 참조 파일만 읽고 체크리스트를 적용한다.
5. **한국형 리스크 확인**: 표시광고, 후기/체험단, 개인정보/픽셀, 업종별 제한을 확인한다.
6. **우선순위 산정**: 영향도, 긴급도, 실행 난이도로 정렬한다.
7. **보고서 작성**: 광고주용 요약과 실무자용 액션을 분리한다.

## Data Mapping

플랫폼마다 지표명이 다를 수 있으므로 아래처럼 통일해서 해석한다.

| Common metric | Korean aliases |
|---|---|
| Spend | 광고비, 집행금액, 비용, 사용금액 |
| Impressions | 노출수, 표시수 |
| Clicks | 클릭수 |
| CTR | 클릭률 |
| CPC | 클릭당 비용, 평균 CPC |
| Conversions | 전환수, 구매수, 상담수, 예약수, 주문수 |
| CPA | 전환당 비용, 구매당 비용, 상담당 비용 |
| Revenue | 매출, 전환매출, 구매금액 |
| ROAS | 광고수익률, ROAS |

계산식:

```text
CTR = Clicks / Impressions
CPC = Spend / Clicks
CVR = Conversions / Clicks
CPA = Spend / Conversions
ROAS = Revenue / Spend
Margin ROAS threshold = 1 / Gross Margin
```

분모가 0이면 억지로 계산하지 말고 `데이터 없음` 또는 `계산 불가`로 표시한다.

## Scoring

100점 만점의 `한국 광고 건강 점수`를 만든다.

| Category | Weight |
|---|---:|
| 추적/전환 데이터 신뢰도 | 25 |
| 예산 낭비와 비효율 | 25 |
| 캠페인/상품/키워드 구조 | 20 |
| 소재와 랜딩/상세페이지 | 15 |
| 플랫폼 믹스와 한국 시장 적합성 | 10 |
| 컴플라이언스 | 5 |

등급:

| Grade | Score | Meaning |
|---|---:|---|
| A | 90-100 | 소폭 최적화 중심 |
| B | 75-89 | 개선 기회 있음 |
| C | 60-74 | 명확한 손실 구간 존재 |
| D | 40-59 | 구조적 문제 가능성 높음 |
| F | 0-39 | 추적/예산/정책 리스크 즉시 점검 |

## Priority Rules

액션은 다음 순서로 정렬한다.

1. **즉시 조치**: 추적 오류, 예산 누수, 정책 위반, 품절/미노출, 3배 이상 CPA 초과
2. **7일 내 조치**: 키워드/상품/타겟 구조 문제, 소재 피로, 랜딩 전환 저하
3. **30일 내 개선**: 실험 설계, 신규 매체 테스트, 리포트 자동화, 장기 구조 개선
4. **보류**: 영향은 작지만 정리하면 좋은 항목

Quick Win 기준:

```text
실행 15분 이내 AND (예산 절감 가능 OR 전환 추적 신뢰도 개선 OR 정책 리스크 감소)
```

## Output Rules

항상 한국어로 작성한다. 광고주가 바로 이해할 수 있는 말과 실무자가 바로 실행할 수 있는 말을 분리한다. 근거 없는 수치 벤치마크를 단정하지 않는다. 최신 플랫폼 정책이나 법률 판단이 필요한 경우 공식 문서 확인 필요성을 표시한다.

보고서에는 반드시 다음을 포함한다.

1. 한 줄 결론
2. 한국 광고 건강 점수
3. 플랫폼별 핵심 문제
4. 즉시 조치 5개
5. 7일 액션 플랜
6. 추가로 필요한 데이터
7. 광고주에게 보낼 수 있는 요약문

## Browser-Assisted Collection

사용자가 "처음 써요", "어디 로그인하면 돼?", "뭘 로그인해야 해?", "링크부터 안내해줘", "로그인된 화면에서 봐줘", "브라우저로 광고 관리자 확인해줘", "화면을 열어둘게"라고 말하면 `references/platform-login-router.md`, `references/first-time-user-guide.md`, `references/browser-assisted-collection.md`를 읽고 진행한다.

브라우저 확인은 항상 읽기 전용이다. 버튼 클릭, 필터 변경, 다운로드, 예산/입찰/상태 변경은 사용자가 명시적으로 요청하고 영향이 명확할 때만 한다. 로그인 정보 입력은 사용자에게 직접 맡기고, 비밀번호/2FA/개인식별정보를 요청하거나 저장하지 않는다.

## Guardrails

- 광고 계정 로그인, 캠페인 수정, 예산 변경을 사용자의 명시 승인 없이 지시하지 않는다.
- 브라우저 사용 시 읽기 전용을 기본값으로 삼고, 저장/수정/삭제/ON-OFF/예산 변경은 하지 않는다.
- 플랫폼 약관이나 보안 정책을 우회하지 않는다. 자동화가 막히면 CSV export 또는 스크린샷 업로드로 전환한다.
- 의료, 금융, 부동산, 건강기능식품, 투자, 교육 결과 보장 표현은 컴플라이언스 리스크로 본다.
- 후기/체험단/인플루언서 광고는 경제적 이해관계 표시 여부를 확인한다.
- 픽셀, SDK, 고객목록, 리타겟팅을 다룰 때 개인정보 처리방침, 동의, 맞춤형 광고 안내를 확인한다.
- Google/Meta 성과를 낮게 보지 않는다. 한국 계정에서는 Google/Meta가 신규 수요 창출과 리타겟팅을 맡고, 네이버/쿠팡이 검색/구매 의도 수요를 맡는 식의 역할 분리가 흔하다.
- 단순히 ROAS가 낮다는 이유만으로 즉시 중단을 권하지 않는다. 마진, 재구매, LTV, 신규 고객 비중을 함께 묻는다.
- 한국 소상공인 계정에서는 운영 복잡도를 낮추는 개선안을 우선한다.
